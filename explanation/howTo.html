Richard Feynman and the difference between knowing the name of something, and knowing something

I started working on this website in an effort to showcase all the different Javascript I’ve been learning lately. However, sometimes we find ourselves just accepting what we learn because we “see that it works”, and we aren’t actually able to explain what it is that we are doing. I am currently about 6 months into learning programming, and as I find myself learning more and more abstract subjects, this is a trap I feel is easy to fall into.

Physicist Richard Feynman was strongly critical of this phenomenon. He argued that we often talk in fact-deficient, obscure generalities, in a way of hiding the fact that we don’t actually know what we’re talking about. To test wether we know what we are actually talking about, or doing, Richard Feynman developed a technique.

There are four steps to the Feynman Technique:

  1. Choose a concept you want to learn about.
  2. Pretend you are teaching it to a student in grade 6.
  3. Identify gaps in your explanation; Go back to the source material, to better understand it.
  4. Review and simplify (optional)

What is JavaScript?

JavaScript is a high-level programming language. Which means that it has a high level of abstraction, from the computer hardware. You can imagine that at the very core of computing, you have programming languages that are meant to communicate directly with the hardware. JavaScript, Python, and other languages are written with actual easy-to-read statements. Take this example describing if a person is old enough to enter.

if (person > 18){
        console.log("Allowed to enter")}
        else {
        console.log("Not old enough")
        }

This is relatively easy to understand, and is close to human language. A low-level language is harder to learn and is closer to directly communicating with the computer in binary code (essentially just ones and zeros). High-level languages are more inefficient, because they don’t fully utilise the computers memory. Languages like JavaScript need to be “compiled” by the computer to function, and essentially gets translated down to binary code, otherwise known as “machine code”. Low-level languages on the other hand need to become disassembled for us to understand them.

JavaScript in particular is useful when we want to add interactivity to websites, or create web based applications, some of which I have attempted to create on this website. All the updated code for this website can be found on the GitHub repo, including the excerpts I have included here.

Get new quote

Built into the homepage, the first button on the website generates a hardcoded quote that gets included into an empty div element on the main HTML file. By creating an onClick function, the div element will include one of the quotes I included in the JavaScript file, as an object. The script identifies the button, and when it gets clicked, it randomises a number which is the length of the quotes object, before using the innerHTML method to change the website.

Remove the line

Incredibly easy, and pretty stupid really. querySelector method to identify the button, getElementById to find the horizontal line. Simply remove it from the website, and add an alert half a second later that tells you it’s removed. Finally, call the function, so it can execute when the website it run.
